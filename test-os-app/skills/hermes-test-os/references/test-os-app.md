# Test OS App Reference

## Routes

| Route | Purpose | Feature Path |
|---|---|---|
| `/foundation` | 5-step foundation wizard | `src/features/test-foundation/` |
| `/validation` | Suite authoring workspace | `src/features/section-validation/` |
| `/verdict` | Run dashboard and reporting | `src/features/verdict-and-reporting/` |
| `/deployment` | EC2/cron setup and status | `src/features/deployment-and-automation/` |

Utility routes (`/runs`, `/inspector`, `/settings`, `/help`) currently render placeholders.

## State Keys

| Key | Owner |
|---|---|
| `test-os.foundation.v1` | `useFoundationState` |
| `test-os.section-validation.v2` | `useSectionValidationState` |
| `test-os.verdict-and-reporting.v1` | `useVerdictState` |
| `test-os.deployment-and-automation.v1` | `useDeploymentState` |

Use reset buttons on each route to restore local demo state.

## Prototype Boundaries

- State is browser-local, not backend persisted.
- Navigation gates are visual-only.
- Hermes runs, EC2 status, and log refresh are represented with local sample data or simulated callbacks.
- `Export Report` opens `/report.html`; the static report file is not generated by the current app.
- Oracle data must remain hidden from build-agent prompts.

## Validation Checklist

After changes:

1. Run `npm run build`.
2. Smoke-check the touched route with `curl -I`.
3. Confirm command labels use `/test-os:*`.
4. Confirm no raw oracle values or hidden fixtures appear in fix prompts or docs.
