## How to Use This

### Option A: Incremental (Recommended)

Build milestone by milestone for better control and review at each step:

1. Copy the `product-plan/` folder to your implementation codebase
2. Start with Shell (`instructions/incremental/01-shell.md`) — gets design tokens and navigation in place
3. For each section milestone:
   - Open `prompts/section-prompt.md`
   - Fill in the section variables at the top (`SECTION_NAME`, `SECTION_ID`, `NN`)
   - Copy/paste into your coding agent
   - Answer clarifying questions, then implement
4. Review and test after each milestone

### Option B: One-Shot

Build the entire app in one session:

1. Copy the `product-plan/` folder to your implementation codebase
2. Open `prompts/one-shot-prompt.md`
3. Copy/paste into your coding agent
4. Answer the agent's clarifying questions about your stack and auth
5. Let the agent plan and implement everything

## Tips

- **Use the pre-written prompts** — they guide the agent to ask about your tech stack and auth before coding
- **Provide `product-overview.md` with every session** — it gives the agent essential context on what Test OS is and why it's adversarially separated from the app repo
- **The components are props-based** — they accept data and fire callbacks; routing, state, and data fetching are your implementation choices
- **Oracle data stays hidden** — Test OS's adversarial value comes from keeping oracle fixtures outside the app repo; make sure your implementation enforces this separation
